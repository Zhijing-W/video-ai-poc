# 糊脸消融实验 · 人脸模型轴（实验设计稿 v2）

> 目标：证明「我们这一系列**人脸模型**（AdaFace / 超分）能在多大程度上解决**糊脸不认 / 不入库 / 认错**」。
> 原则：**分桶、建档、判定全部走 demo 同一套产品代码**，不搞"测一套逻辑、demo 另一套逻辑"。

---

## 1. 范围（人脸 + 人形，ArcFace 基线，递进消融）

- **ArcFace 人脸为基线**，逐步叠加：AdaFace（新人脸损失）/ 超分（人脸增强）/ **人形 ReID（多模态）**。
- **必须含人形**：因为"**不入库**"和"**无脸 none 桶**"人脸救不了，只有人形能补建档、补认出——这正是要证明的核心。
- **步态**需走路**连续帧序列**，单图数据集（Market/ChokePoint）**无法评测** → 不进本实验，已装进 demo，单独用视频/步态集（CASIA-B / GREW）评。
- 关于"ArcFace 2"：**无官方 v2**；针对糊脸的后继就是 **AdaFace**（已在 arm）；要更强人脸可选换骨干 **antelopev2**（WebFace42M，作可选 B5）。

## 2. arm 设计（多模态递进，ArcFace 基线）

| arm | 人脸后端 | 超分 | 人形 ReID | 说明 |
|---|---|---|---|---|
| **B0** baseline | ArcFace (buffalo_l/w600k_r50) | ✗ | ✗ | 基线 |
| **B1** | AdaFace (ir_101) | ✗ | ✗ | 换新人脸损失（低清脸更强）的增益 |
| **B2** | ArcFace | ✓ + **重评质量** | ✗ | 超分（拉清再识别、拉清后重评质量）的增益 |
| **B3** | ArcFace | ✗ | **✓ OSNet** | **加人形**：救 none/背身 + 补建档（不入库） |
| **B4** full | AdaFace | ✓ | ✓ | 全栈天花板 |
| *(可选)* **B5** | **antelopev2** | ✗ | ✗ | 更强骨干 vs ArcFace（更大训练集） |

> 超分**默认"糊才超分"门控**（对齐 demo：`blur_var<60 且 |yaw|<80 且够大` 才超），不是无脑全超。
> 人形融合走产品 `fuse_identity` 的**软性连续加权**（人脸质量低→人脸降权→人形主导）。

## 3. 与 demo 对齐的三处关键逻辑（不能跑偏）

### 3.1 质量分桶 = 产品 `assess_quality`（已对齐）
多层硬门控：**角度**(yaw≥80 / pitch 低头≤-35 严、抬头≥50 宽) + **模糊**(拉普拉斯<15 或 det<0.5) + **尺寸**(太小) →
任一层不合格 = `poor`；全清晰且够大 = `clear`；其余 `marginal`；检不到脸 = `none`。
> 人脸轴主看 **poor / marginal**（检出但糊）；**none 桶人脸无解**（记录、留给人形/步态），不计入人脸轴的"能救回"。

### 3.2 建档 = demo 真门控（本次要补上，别再手挑清晰图）
- **人脸库**：`auto_enroll = (category == "clear")` —— **只有 clear 脸才建档当锚点**；marginal/poor/none **查但不建**（防污染）。
- 一个身份若**从没出现过 clear 脸** → **人脸建不了档（不入库）** → 该身份的 probe 全部计为"人脸认不出"（这正是要量化的"不入库"代价）。
- **超分后重评（A2/A3）**：在**修复后的图**上重算 `category` 再决定入库（对齐业界 FIQA 最佳实践：质量测输出、不测输入）。→ 拉清后够格 `clear` 的脸可以建档 → 可能提升覆盖率。

### 3.3 判定 = 开集 + 产品置信度阈值（本次要补上，别再闭集强制猜）
- probe 里**混入库外冒充者**（distractor 身份）；用阈值判 **命中 / 拒识**，而不是无脑取 top-1。
- 三种结局：**命中且正确** / **命中但错人（认错）** / **正确拒识**。

## 4. 指标（三个目标 → 三个指标 → 哪个 arm 上升/下降来证明）

| 目标 | 指标 | 定义 | 证明方向 | 主要区分 |
|---|---|---|---|---|
| **糊脸不认** | **差脸桶识别率** TPIR@rank1 | poor/marginal 桶上，命中且正确的比例 | ↑ | A1/A2/A3 vs A0（backend + 超分都可能帮） |
| **不入库** | **入库覆盖率** | 按真门控能成功建档的身份数 / 总身份数 | ↑ | 主要由**超分+重评**区分：A0=A1（不改 category），A2/A3（拉清→更多 clear→覆盖↑） |
| **认错** | **误识率 FMR** + **正确拒识率** | 开集下：错认已知 + 误接受冒充者 = 误识；正确判"不在库" = 拒识 | FMR↓ / 拒识↑ | A1(AdaFace 质量自适应→糊脸低分→少错匹配) vs A0；超分是否幻觉→误识是否↑ |

> 补充产出：**识别率 vs 误识率的 DET/权衡曲线**（扫阈值）——一眼看出每个 arm 在"多认对"与"少认错"上的取舍。

**关键洞察（务必写进结论）**：
- `category`（建档门控）只看**检测/模糊/角度/尺寸**，**与识别后端无关** → **ArcFace↔AdaFace 不改入库覆盖率**；只有**超分+重评**改覆盖率。
- 所以"覆盖率"这条主要回答**"超分能不能救回'不入库'"**；"识别率/误识率"回答**"AdaFace/超分能不能救回'不认/认错'"**。

## 5. 数据与协议

- **主力**：Market-1501（脸普遍 tiny/糊/侧/无脸 → 天然出 poor/none 难例）。
- **对照**：ChokePoint（近正脸清晰 → 出 clear 对照桶）。
- **开集**：`--distractors N`（额外身份只入库/作冒充者，不作正样本 probe）→ 候选池放大、可算 FMR。
- **划分**：每身份"最清晰若干帧"过**真门控**建档（clear 才入库）；其余帧作 probe；图像互斥不泄漏。
- **阈值**：用产品 `face_hit_thresh` / `IDENTITY_RESOLVE_THRESH`，或扫阈值出曲线。

## 6. 预期结果与风险（诚实预判）

1. **Market 上 AdaFace/超分可能"平甚至负"**：AdaFace 擅长"低清但**完整正脸**"，Market 是 **tiny+侧脸** → 上次 A1 反降(66→55)。超分门控在 tiny-sharp 脸上也常不触发。
   → **要看到人脸模型的正向价值，需真实中远距糊脸集（SCFace / 真实监控）**；Market 结论更多是"管线跑通 + 反例证据"。
2. **GFPGAN 可能幻觉伤识别**：拉清后 `category` 变 clear、覆盖率↑，但幻觉出的身份可能**增加误识** → 必须用 **FMR 把关**，别只看覆盖率。若 A2/A3 覆盖↑但误识↑ → 证据指向"**换 CodeFormer（保真度可调）或改用 AdaFace 顶替超分**"。
3. **none 桶（无脸）**：人脸 arm（B0/B1/B2）一律救不回 → 正是 **B3/B4 加人形**要救的地方；none 桶识别率从"人脸≈0"到"加人形后大幅回升"，就是多模态价值的最直接证据（步态能进一步补背身，但需序列，另做）。
