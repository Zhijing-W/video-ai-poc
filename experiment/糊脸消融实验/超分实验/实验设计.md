# 人脸超分门控实验

> 上一轮实验未使用 Train 集校准 CR-FIQA，且聚合了每条轨迹的多张人脸，与产品只处理最佳帧的逻辑不一致，因此结果仅作诊断，需按本协议重跑。

## 1. 实验目标

比较三种处理方式对 ArcFace 识别效果和运行成本的影响：

| 方案 | 处理方式 |
|---|---|
| A 原图 | 固定对齐脸直接调用 ArcFace |
| B 全量超分 | 所有 `can_match=true` 的脸先调用 GFPGAN，再调用 ArcFace |
| C 产品门控 | 仅 `can_superres=true` 的脸调用 GFPGAN，其余使用原图 ArcFace |

三组必须使用相同人物、Gallery、Query、对齐脸、ArcFace 和产品 `FACE_HIT_THRESH`。

## 2. Train 集校准质量门控

CR-FIQA 输出是原始回归分数，不是通用的 0–1 概率。正式测试前，先在 MEVID Train 集完成：

```text
SCRFD检测与五点对齐
→ CR-FIQA原始分数
→ ArcFace识别可用性
→ 校准FACE_FIQA_POOR_THRESH / FACE_FIQA_CLEAR_THRESH
→ 确定clear / marginal / poor及can_superres规则
```

校准完成后冻结所有质量阈值。Test 集不得再次调参。

## 3. 按产品逻辑冻结 Test 输入

当前 Event Monitor 对每条人物轨迹只处理一个最佳帧：

```text
轨迹全部帧
→ 使用产品人形质量公式选择best_idx
→ 只在best_idx运行SCRFD和人脸质量评估
→ 固定唯一112×112对齐脸
```

实验必须复用相同逻辑，不能从多张脸中额外挑选最有利的人脸，也不能聚合8张人脸 embedding。
正式运行默认扫描整条轨迹；`--frames-per-track`非0只用于调试抽样，不能用于正式结果。

`prepare` 阶段只运行检测、对齐和质量评估，不调用 ArcFace、GFPGAN 或 3D 几何。固定保存：

```text
sample_id / identity / gallery-query角色
产品选择的best_idx
原始112×112对齐脸及SHA256
category / defects / can_match / can_superres
FIQA before
```

没有检测到脸的轨迹记为 `none`，仍保留在最终测试集中。

## 4. 固定输入上运行 A/B/C

```text
A = ArcFace(original)
B = ArcFace(GFPGAN(original))                 if can_match else none
C = ArcFace(GFPGAN(original))                 if can_superres
    ArcFace(original)                         otherwise
```

约束：

- Gallery 只使用 A 原图建立一次。
- A/B/C 使用同一批最终 Query。
- 统一使用产品 `FACE_HIT_THRESH`，不做 FMR 阈值扫描。
- GFPGAN 失败时 B/C 回退 A，不删除样本。
- 超分后可以重新计算 FIQA，但不能改变原始质量桶和门控结果。

## 5. 身份划分

- 默认 27 个建档身份。
- `--imposter-subjects -1` 使用建档后全部剩余可用 Test 身份作为陌生人。
- 身份划分与随机种子固定，并写入结果 JSON。

## 6. 报告指标

- Rank-1。
- 固定 `FACE_HIT_THRESH` 下的已知人物正确接受率。
- 陌生人误接受率。
- 错误→正确、正确→错误。
- 陌生人误接受增加/减少。
- FIQA before/after。
- 原图与超分后 ArcFace embedding 余弦。
- GFPGAN 调用次数、成功次数、总耗时和平均耗时。

## 7. 与完整产品实验的关系

本实验只验证人脸超分的净作用，不调用人形 ReID 和步态。确定 A/B/C 中的最佳人脸方案后，再接入 `run_mevid_e2e.py`，使用产品人脸、人形和步态完整链路验证最终效果。
